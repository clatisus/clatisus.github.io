  1. k路归并

现有$k$个有序数组（从小到大排序），每个数组中包含$n$个元素。你的任务是将他们合并成$1$个包含$kn$个元素的有序数组。首先来回忆一下课上讲的归并排序算法，它提供了一种合并有序数组的算法$Merge$。如果我们有两个有序数组的大小分别为$x$和$y$，$Merge$算法可以用$O(x+y)$的时间来合并这两个数组。

  • 如果我们应用$Merge$算法先合并第一个和第二个数组，然后由合并后的数组与第三个合并，再与第四个合并，直到合并完k个数组。请分析这种合并策略的时间复杂度（请用关于$k$和$n$的函数表示）。

  • 针对本题的任务，请给出一个更高效的算法，并分析它的时间复杂度。（提示：此题若取得满分，所设计算法的时间复杂度应为$O(nklogk)$）。

（1）每次归并相邻两路，第一次归并，两个大小为 $n$ 的数组合成大小为 $2n$ 的数组，时间复杂度为 $O(2n)$，第二次归并，刚才合成的大小为 $2n$ 数组与第三个大小为 $n$ 数组合并，时间复杂度为 $O(3n)$，以f此类推，可以得到如下计算公式：
$$
T_k(n) = 2n + 3n + \cdots + kn=O\left(n\frac{(1
+k)k}{2}\right) =O(nk^2)
$$

(2) 设当前对第 $l$ 路到第 $r$ 路归并排序，我们先递归对 $[l, m]$ 和 $[m + 1, r]$ 路归并排序（$m = \frac{l + r}{2}$）, 再将二者的排序结果进行归并排序，可得如下时间复杂度计算公式：

$$
T_n(k) = 2T_n(k/2) + O(kn)
$$

在计算过程中，可以把 $n$ 当成常数，计算类似归并排序的时间复杂度计算，化简可以得到：

$$
T_n(k) = O(nk\log k)
$$

  1. 战队补给

现有$2^l$个堡垒组成一条战线，编号为$1,2,3,\cdots,2^l$。其中每个堡垒都可能有一个或多个士兵驻扎，也可能没有任何士兵驻扎。已知士兵共有$n$个，且第$i$个士兵驻扎在编号为$a_i$的堡垒中。现请你用最小的费用给整条战线提供补给（即是说，为区间$[1,2^l]$中的所有堡垒提供补给）。为战线$[1,2^l]$提供补给可以按照下述两种方式进行：

  • 若当前区间组成的战线中还剩余至少两个堡垒，可以将该区间均分为左右两段，并分别为其提供补给。所需的总费用为补给这两段战线的费用之和。

  • 直接为当前区间组成的战线提供补给。若这段战线中的所有堡垒均没有任何士兵驻扎，则所需的费用为$A$；否则，费用为$num\timeslen\timesB$。其中$A, B$为给定的常数，$num, len$分别为这段战线中士兵的总数以及堡垒的总数。

例如，现有四个堡垒，编号为$1,2,3,4$。只有一个士兵，其驻扎在2号堡垒。可选的补给方式有多种：一种方式为直接为整条战线$[1,4]$提供补给，所需费用为14B，其中1表示当前中共有1名士兵，4表示当前区间中共有4个堡垒；另外一种可行的方式是将该区间平均分为两段$[1,2]$和$[3,4]$。并分别为这两段提供补给，和第一种方式类似，区间$[1,2]$所需的代价为$1\times2\timesB$，其中$1, 2$分别表示当前区间的士兵个数和堡垒个数。而区间$[3,4]$由于没有任何士兵驻扎，所需的费用为$A$。因此，这种补给方式的总费用为$A+2B$。当然也可以将区间$[1,2]$或$[3,4]$继续均分并分别进行补给。请设计一个高效的算法计算为整条战线体提供补给所需的最小费用，并尽可能准确地分析该算法的时间复杂度

若计算区间 $[l, r]$ 中的补给，递归计算 $[l, m]$ 和 $[m + 1, r]$ 的结果，并将他们相加，假设结果为 $C$。然后计算如果不递归的代价，如果 $[l, r]$ 中士兵数量为 $0$，则取 $\min{A, C}$；否则，取 $\min{\text{num} * (r - l + 1) * B, C}$。

时间复杂度计算公式为：
$$
T(2^l)=2T(2^{l-1})+O(1)=O(2^l)
$$

  1. 区间计数

给定一个包含n个元素的数组$A= [a1, a2,\cdots, an]$。对数组A中的任意区间$[l, r](1\lel\ler\len)$，该区间的和可表示为$S[l,r]=\sum_{i=l}^r ai$。请设计一个高效的分治算法统计有多少个区间$[l, r]$满足：$X\leS[l,r]\leY$（$X, Y$为给定的常数）。并分析该算法的时间复杂度。

统计满足 $X \leq S_{[l, r]} \leq Y$ 的区间 $[l, r]$ 的个数，也即 $\sum_{1\le l \le r \le n} [X \le S_{[l,r]} \le Y]$，设前缀和函数为 $S(x) = \sum_{i = 0}^{x}a_i$，则所求为

$$
\sum_{1\le l \le r \le n}[X \le S(r) - S(l-1) \le Y] = \sum_{0 \le l < r \le n}[X \le S(r) - S(l) \le Y]
$$

我们考虑 $L \leq l < r \leq R$ 的时候，合法的二元组 $(l, r)$ 的对数，现在求区间 $[L, R]$ 对答案的贡献。

在区间 $[L, R]$ 中，设 $M = \frac{L + R}{2}$ 递归出区间 $[L, M]$ 和 $[M + 1, R]$ 的答案（也即 $L \leq l < r \leq M$ 和 $M + 1 \leq l < r \leq R$ 时的答案）。那么还剩下 $L \leq l \leq M, M + 1 \leq r \leq R$ 的答案，也即跨左右区间的答案。

设左右区间的 $S$ 函数已经升序排序，我们枚举 $l \in [L, M]$，对合法的 $r$ 有：

$$
\begin{aligned}
&X \le S(r) - S(l) \le Y \
&S(l) + X \le S(r) \le S(l) + Y
\end{aligned}
$$

区间 $[S(l)+X, S(l)+Y]$ 都是 $r$ 的合法区间。我们用双指针 $i, j$ 来表示这个区间，当 $l$ 增加的时候，$S(l) + X$ 和 $S(l) + Y$ 都会 不降，我们向右移动 $i, j$ 即可得到 $r$ 的新的合法区间，最后将区间归并排序，复杂度为 $O(n\log n)$（$i, j$ 都只会增加不会减少）。

  1. 向量的最小和问题

给定$n$个二维向量$v1,v2,\cdots ,vn$。每一个向量$vi= (xi, yi)$都可以变换为如下四种形式：

1.$v^1_i= (xi, yi)$

2.$v^2_i= (-xi, yi)$

3.$v^3_i= (xi, -yi)$

4.$v^4_i= (-xi,-yi)$

请你设计一个高效的算法从$n$个向量中找出两个向量，使得他们以某种形式相加后的模长最小。换言之，请找出两个向量$vi, vj(1\le i, j\le n$且$i\neq j)$，以及两个整数$k1, k2(1\le k1, k2 \le 4)$，使得$||v^{k1}_i+v^{k2}_j||^2$最小。此外，请分析该算法的时间复杂度

注意到在此问题中，向量相加等价于向量相减（适当调整符号即可），所以等价于求最近的点对之间的距离（每个点可以在四个象限出现）。再注意到两点的距离公式：

$$
\sqrt{(x_i-x_j)^2+(y_i-y_j)^2}=\sqrt{x_i^2+x_j^2+y_i^2+y_j^2-2x_ix_j-2y_iy_j}
$$

那么显然最小值一定是在 $x_i, x_j$ 同号且 $y_i,y_j$ 同号的时候取到。我们不妨另所有的坐标都为非负数。所以原问题等价于，对于 $n$ 个二维平面上的点 $p_1,p_2,\dots,p_n$。找出两个点 $p_i, p_j(1\leq i, j \leq n, i\neq j)$，求 $\min \sqrt{(x_i-x_j)^2+(y_i-y_j)^2}$。算法如下：

我们先对 $n$ 个点按照 $x$ 坐标升序排序并重新标号，然后递归求解。对于区间 $[l, r]$，令 $m=\frac{l+r}{2}$，记录当前区间的中点 $M=p_m$，然后递归区间 $[l,m]$ 和 $[m+1,r]$，并在回溯的时候按 $y$ 坐标归并排序。设求得的两个区间内部的答案分别为 $h_1, h_2$，令 $h=\min {h_1, h_2 }$。现在考虑跨左右区间的点对中，可能更新答案的点对（有可能距离小于 $h$ 的点对）。

令 $S = {p_i|i \in [l, r], |x_i-M_x|<h}$，我们只用在 $S$ 集合中取寻找可以更新答案的点对即可。对于 $S$ 中的每个点 $p_i$，我们令：

$$
C(p_i) = {p_j|p_j\in S, y_j\leq y_i, |y_i-y_j|<h}={p_j|p_j\in S, y_i-h<y_j\leq y_i}
$$

因为递归回溯的时候已经按照 $y$ 坐标排序，所以当我们枚举的 $p_i$ 发生变化的时候，集合 $C(p_i)$ 很容易求得（我们只需要移动合法的 $j$ 的区间左端点即可）。然后我们暴力的将 $p_i$ 和 $C(p_i)$ 集合中的每个点求距离更新答案即可。

时间复杂度证明：

对于每个 $p_i$，$C(p_i)$ 中所有点的 $y$ 坐标范围为 $(y_i-h, y_i]$，且 $C(p_i) \cup {p_i}$ 中所有点的 $x$ 坐标范围为 $(M_x-h, M_x+h)$，这构成了一个 $2h\times h$ 的矩形。我们将其拆分为两个 $h\times h$ 的正方形 $(M_x-h, M_x]\times (y_i-h, y_i]$ 和 $[M_x, M_x+h)\times (y_i-h, y_i]$。

显然每个正方形内部的任意两点间距离都大于等于 $h$（因为这两点一定来自同一个递归，否则 $h$ 不是最小值），再将每个 $h\times h$ 的正方形拆成四个 $\frac{h}{2}\times \frac{h}{2}$ 的小正方形，可以发现每个小正方形内部最多有一个点（否则如果有两个点，最大距离为对角线 $\frac{h}{\sqrt 2}$，而 $\frac{h}{\sqrt 2} < h$）。

由此 $|C(p_i)| \leq 8$，总的复杂度为 $O(n\log n)$。
